Loop Prevention Demo

Demonstrates how the MQTT bridge prevents message loops in bidirectional configurations using fingerprint-based duplicate detection.

How Loop Prevention Works:
  • Fingerprinting: Each message is hashed (topic + payload + QoS + retain) to create a unique fingerprint
  • TTL-based caching: Fingerprints are cached for a configurable time window (TTL)
  • Duplicate blocking: If the same fingerprint appears within the TTL window, the message is blocked
  • TTL expiration: After TTL expires, the same message can pass through again
Demo Instructions:
  1. Click "Send Single Message" - message passes through to Broker B
  2. Click "Send 10x Rapid Fire" - only the first message passes, 9 are blocked
  3. Wait for TTL to expire, then send again - message will pass through
