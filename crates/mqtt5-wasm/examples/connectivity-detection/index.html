Browser Connectivity Detection Demo

Demonstrates how the WASM MQTT client detects browser online/offline state and pauses reconnection when the network is unavailable.

How it works:
  • on_connectivity_change(callback) fires when the browser goes online or offline
  • is_browser_online() returns the current network state synchronously
  • Reconnection pauses while offline and resumes immediately when the network returns
  • Backoff resets after a network outage (the failure was connectivity, not server rejection)

To test connectivity detection:
  1. Click "Connect to Broker" to start the MQTT connection
  2. Toggle your OS Wi-Fi off, use airplane mode, or use the DevTools Network "Offline" throttle
  3. Watch the event log: reconnection pauses while offline
  4. Restore connectivity: reconnection resumes immediately from attempt 0
