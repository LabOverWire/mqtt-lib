Load Balancer Redirect Demo

This demonstrates MQTT v5 Server Redirect using the UseAnotherServer reason code. An in-browser WASM load balancer redirects clients to one of two native backend brokers running on localhost.

Architecture: WASM Client → WASM LB Broker (CONNACK 0x9C + ServerReference) → Client reconnects via WebSocket to native Backend A or B
Prerequisites: Start two native brokers with WebSocket enabled before opening this page: 
mqttv5 broker --host 0.0.0.0:1884 --ws-host 0.0.0.0:9001 --allow-anonymous
mqttv5 broker --host 0.0.0.0:1885 --ws-host 0.0.0.0:9002 --allow-anonymous
Try this: Click "Connect 4 Clients via LB" to see clients get distributed across the two native backends. Then use the CLI to cross-verify: mqttv5 sub -t demo/# --host 127.0.0.1 --port 1884 --non-interactive
