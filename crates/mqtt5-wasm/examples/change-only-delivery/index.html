Change-Only Delivery Demo

This demonstrates the change-only delivery feature where the broker only delivers messages when the payload differs from the last delivered value.

How Change-Only Deliverys Work:
  • The broker tracks the last payload hash per topic per subscriber
  • When a new message arrives, it's only delivered if the payload changed
  • This reduces bandwidth for sensors that frequently publish unchanged values
  • Configured via topic patterns: sensors/#
Try this: Click "Publish Same Value" multiple times - notice only the first message is received!
Then click "Publish Different Value" to see a new message delivered.
